# 個人 VIP 系統架構規劃（V1）

> 依據需求拆分為「會員端」、「後台端」、「資料與流程」、「權限與通知」、「迭代路線圖」。

## 1. 產品目標與範圍

- 建立可「依會員標籤個人化展示」的 VIP 服務入口。
- 整合會員消費、曬單、跟團等行為，做進度與任務管理。
- 讓後台可快速完成標籤管理、課程投放、公告維護、報名追蹤與提醒。

---

## 2. 會員端資訊架構（IA）

### 2.1 首頁模組

1. **VIP 進度顯示區**
   - 顯示目標（例：累積消費達 30,000、曬單 10 次）。
   - 顯示進度條、目前跟團次數、目前累積金額。
   - 顯示「距離下個門檻」差距（可提升完成率）。

2. **會員專屬活動區**
   - 依會員標籤展示活動（打卡、抽獎、專屬任務）。

3. **近期線上課程區**
   - 展示「進行中 / 即將開始」課程。
   - 顯示講師、時間、主題、報名狀態。

4. **近期開團預告區（共用行事曆）**
   - 所有會員看到相同開團行事曆（日期、商品名、體驗團/中年團）。
   - 可導連至官網活動頁（單一資訊來源）。

5. **打卡系統入口**
   - 只顯示會員已購買商品對應的打卡活動。

6. **會員任務專區**
   - 可上傳任務佐證照片（例如社群貼文截圖）。
   - 上傳後進入「待審核」，審核通過後發積分。

7. **公告區**
   - 顯示粉絲福利、活動異動、系統通知。

---

## 3. 後台管理架構

### 3.1 會員管理與資料串接

- **自動匯入欄位**：消費金額、累積金額、曬單次數、跟團次數（來源：顧客資料表）。
- **手動維護欄位**：姓名、電話、會員等級、備註。
- **操作能力**：
  - 會員查詢/篩選/分群。
  - 標籤新增、刪除、套用。
  - 依標籤發放活動資格。

### 3.2 標籤管理系統

- 標籤可代表商品、需求、健康目標、生命周期階段。
- 支援兩種套用方式：
  1. 一鍵勾選（例如曾購買「益生菌」者全部套標）。
  2. 手動指定（顧問服務情境）。
- 標籤驅動個人化：課程、活動、公告、任務可設定目標標籤。

### 3.3 課程管理與投放

- 課程欄位：名稱、封面、時間、講師、大綱、報名上限、提醒設定。
- 投放規則：
  - 依標籤投放（如瘦身標籤 -> 瘦身課程）。
  - 可額外手動增減名單。
- 課程可綁定「課程標籤」（例如 5/1 益生菌課程）。

### 3.4 課程活動專區與報名

- 會員端提供「我要報名」。
- 後台可查看：報名名單、出席狀態、提醒發送狀態。
- 每日早晨排程輸出當日/近期課程報名清單。

### 3.5 公告管理

- 富文字編輯、上下架時間、可見對象（全體/特定標籤/VIP等級）。

## 4. 建議技術架構（實作導向）

## 4.1 分層

- **前台（會員端）**：Web App（RWD）
- **後台（Admin）**：管理介面
- **API 層**：會員、標籤、任務、課程、公告、通知
- **排程/工作佇列**：每日彙整、提醒發送、資料同步
- **資料層**：主資料庫 + 檔案儲存（任務照片）

## 4.2 主要服務模組

1. `member-service`：會員資料、VIP 進度計算。
2. `tag-service`：標籤 CRUD、批次套標。
3. `campaign-service`：活動/打卡/抽獎規則。
4. `course-service`：課程管理、報名。
5. `task-review-service`：任務上傳、審核、積分發放。
6. `notice-service`：公告管理。
7. `notification-service`：Email/LINE 提醒。
8. `sync-service`：顧客資料表同步（ETL/API）。

---

## 5. 核心資料模型（簡版）

- `members`：會員主檔（姓名、電話、等級、狀態）。
- `member_metrics`：累積消費、跟團次數、曬單次數、最後同步時間。
- `vip_rules`：VIP 門檻規則（消費門檻、曬單門檻、生效區間）。
- `member_vip_progress`：會員對規則的即時計算結果。
- `tags` / `member_tags`：標籤與會員關聯。
- `courses` / `course_targets` / `course_registrations`：課程與目標族群與報名。
- `tasks` / `task_submissions` / `task_reviews`：任務與上傳與審核紀錄。
- `announcements`：公告。
- `checkin_campaigns`：打卡活動定義。
- `notification_logs`：通知發送結果（Email/LINE）。

---

## 6. 關鍵流程（E2E）

### 6.1 會員進度更新

1. `sync-service` 每日/每小時拉取顧客資料表。
2. 寫入 `member_metrics`。
3. `member-service` 依 `vip_rules` 重算 `member_vip_progress`。
4. 前台首頁即時讀取進度條資料。

### 6.2 任務上傳與審核發分

1. 會員上傳截圖 -> `task_submissions` 狀態為 `pending`。
2. 管理員審核 `approve/reject`。
3. `approve` 觸發積分/獎勵發放。
4. 會員收到通知與結果。

### 6.3 課程投放與提醒

1. 管理員建立課程並指定目標標籤。
2. 會員端顯示符合條件課程。
3. 會員點擊報名，寫入 `course_registrations`。
4. 每日早上排程產生提醒名單並推送 Email/LINE。
5. 開課當天再次提醒。

---

## 7. 權限與審計

- 角色：`super_admin`、`operator`、`reviewer`。
- 權限重點：
  - 標籤批次操作、任務審核、課程發布均需記錄操作人。
  - 所有關鍵操作寫入 `audit_logs`（前後值、時間、IP、操作者）。

---

## 8. KPI 與成效追蹤

- VIP 達標率、任務完成率、課程報名率、課程出席率。
- 標籤轉化（投放後購買率/回購率）。
- 通知送達率與開信/點擊率（Email）或已讀率（LINE）。

---

## 9. 建議開發里程碑（MVP -> V2）

### MVP（4~6 週）
- 會員資料同步、VIP 進度、標籤管理。
- 課程管理 + 報名 + 每日提醒。
- 任務上傳 + 後台審核。
- 公告管理。

### V1.5
- 打卡活動規則引擎。
- 抽獎活動與中獎發放。
- 官網開團行事曆雙向同步。

### V2
- 自動化推薦（依標籤/行為）。
- A/B 測試投放與進階報表。
- 顧客分群預測（流失/回購）。

---

## 10. 風險與注意事項

- 顧客資料來源品質（欄位缺漏、主鍵不一致）需先治理。
- LINE/Email 通知需處理退訂、失敗重送與法規遵循。
- 任務照片含個資，需設定保存期限、遮罩與權限控管。
- 標籤濫用會導致投放失準，需建立命名與生命週期規範。

```bash
cd /workspace/kewiVIP
python3 -m http.server 8080
```

開啟：

- `http://localhost:8080/web/index.html`
- `http://localhost:8080/web/admin.html`
